# 🚀 运行打包脚本指南

## 问题说明

传统的 `.bat` 批处理文件在Windows上可能显示中文乱码，因为编码问题。

## ✅ 解决方案（3选1）

### 方案1：使用PowerShell脚本（推荐）⭐

**优点：** 完美支持中文，带彩色输出

**使用方法：**
1. 右键点击 `package_for_github.ps1`
2. 选择"使用PowerShell运行"

或在PowerShell中执行：
```powershell
cd D:\tool\pdata\LLMs\customer_service_agent
.\package_for_github.ps1
```

**如果遇到"无法加载文件"错误：**
```powershell
# 以管理员身份运行PowerShell，执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

---

### 方案2：使用英文版BAT文件

**文件名：** `package_for_github_fixed.bat`

**优点：** 无乱码问题，兼容性好

**使用方法：** 双击运行即可

---

### 方案3：手动执行命令

如果脚本都无法运行，可以手动执行以下命令：

```cmd
REM 进入项目目录
cd D:\tool\pdata\LLMs\customer_service_agent

REM 初始化Git仓库
git init

REM 添加所有文件
git add .

REM 创建提交
git commit -m "Initial commit: Intelligent Customer Service System"

REM 关联GitHub仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/仓库名.git

REM 设置主分支
git branch -M main

REM 推送到GitHub
git push -u origin main
```

## ❓ 常见问题

### Q1: PowerShell脚本无法运行？
**A:** 需要修改执行策略：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q2: BAT文件中文乱码？
**A:** 使用以下任一方法：
- 使用 `package_for_github.ps1` (PowerShell版本)
- 使用 `package_for_github_fixed.bat` (英文版本)

### Q3: 提示Git未安装？
**A:** 
1. 下载Git: https://git-scm.com/download/win
2. 安装后重启命令行
3. 或使用GitHub Desktop

### Q4: src/new_agent.py不存在？
**A:** 文件已经在正确位置了，可以直接运行脚本。
